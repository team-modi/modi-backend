# (시스템) 카탈로그 동기화 — 목록+상세 한 패스

> 시나리오 2.10 — 시스템이 외부 공공데이터 전시 API를 수집해 DB에 적재한다. **목록과 상세를 한 패스로 받아 적재 시점에 곧바로 완전한 행**을 만든다(별도 상세 백필 잡 없음). 부팅 직후 1회(cold start 방지), 이후 매일 자정에 재동기화하며 신규분 장르 분류(07 문서)도 이때 이어서 수행한다.

**다이어그램이 필요한 이유**
- 3갈래 적재 판정: 신규는 상세까지 채워 insert / 기존 미완성 행은 상세만 채워 완성 / 이미 완성된 행은 외부 호출 없이 건너뜀
- 데이터 보존 규칙: 상세 외의 보강값(장르 등)은 재적재로 덮지 않는다 — 전량 완성된 정상 상태에선 값이 안 바뀐다
- 트랜잭션 설계: 루프는 트랜잭션 밖, 행 단위 save만 각자 트랜잭션 — 수백 건 상세 API 호출을 한 트랜잭션에 물지 않는다(커넥션 장기 점유 방지)
- 실패 격리: 기간 비정상은 스킵, 상세 호출 일시 실패는 그 행만 연기 — 단건이 배치 전체를 중단시키지 않는다

```mermaid
sequenceDiagram
    actor 스케줄러
    participant ExhibitionFacade
    participant CultureExhibitionClient
    participant ExhibitionRepository
    participant Exhibition

    Note right of 스케줄러: 부팅 1회 ExhibitionCatalogBootSync(데몬 스레드) / 매일 자정 ExhibitionSyncScheduler.syncDaily

    스케줄러->>ExhibitionFacade: 카탈로그 동기화
    ExhibitionFacade->>CultureExhibitionClient: 전체 목록 수집 (페이지 순회)
    alt 인증키 미설정
        CultureExhibitionClient-->>스케줄러: 외부 호출 없이 0건 종료
    end
    alt 외부 API 통신 실패
        CultureExhibitionClient-->>스케줄러: 실패 (경고 로그 후 다음 주기 재시도)
    end

    loop 수집 목록 건별 (루프는 트랜잭션 밖)
        alt 기간 비정상 (종료일 < 시작일)
            ExhibitionFacade->>ExhibitionFacade: 스킵 (엔티티 생성 전에 걸러 dirty-flush 방지)
        else 정상
            ExhibitionFacade->>ExhibitionRepository: externalId로 기존 행 조회

            alt 기존 행이 이미 상세까지 완성
                ExhibitionFacade->>ExhibitionFacade: 건너뜀 (상세 API 호출 없음 — 값 무변경)
            else 기존 행이나 상세 미완성
                ExhibitionFacade->>CultureExhibitionClient: 단건 상세 조회
                ExhibitionFacade->>Exhibition: 상세 반영 또는 확인 완료 표기
                ExhibitionFacade->>ExhibitionRepository: 저장 (행 단위 트랜잭션)
                Note right of ExhibitionFacade: 상세만 채워 완성 — 장르 등 다른 보강값은 무변경
            else 신규 (externalId 미존재)
                ExhibitionFacade->>Exhibition: 목록 필드로 CATALOG 전시 생성
                ExhibitionFacade->>CultureExhibitionClient: 단건 상세 조회
                alt 상세 수신
                    ExhibitionFacade->>Exhibition: 상세 필드 반영 + 동기화 시각 기록
                else 원천에 상세 없음 (빈 응답)
                    ExhibitionFacade->>Exhibition: 확인 완료 표기 (값은 채우지 않음)
                    Note right of Exhibition: 재조회 대상에서 제외 — 매 주기 반복 호출 방지
                end
                ExhibitionFacade->>ExhibitionRepository: 저장 (행 단위 트랜잭션)
            end

            alt 상세 조회 등 일시 실패 (예외)
                ExhibitionFacade->>ExhibitionFacade: 이 행만 연기 (경고 로그 후 다음 행 계속)
                Note right of ExhibitionFacade: 불완전한 행을 적재하지 않고 다음 주기에 재시도
            end
        end
    end

    ExhibitionFacade-->>스케줄러: 신규 적재 건수 반환 (기존 행 상세 완성 건은 제외)
    Note right of ExhibitionFacade: 로그엔 수집·신규적재·기존상세완성·기간스킵·실패연기 집계
```
