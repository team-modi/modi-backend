# 전시 상세 조회

> 시나리오 2.3 — 사용자가 특정 전시의 상세 정보를 조회한다. 상세 진입 시 조회수가 1 증가하고, 상세가 아직 안 채워진 CATALOG 전시는 진입 때 외부 상세를 1회 지연수집해 캐시한다.

> 지연수집은 이제 **예외 경로**다 — 카탈로그 동기화(06 문서)가 적재 시점에 상세까지 채우므로, 동기화가 그 행을 아직 완성하지 못한 경우(직전 주기에 상세 호출이 실패해 연기됨)에만 탄다.

**다이어그램이 필요한 이유**
- 조건 분기: 전시 없음(404), 타인의 CUSTOM 접근(403), 상세 미수집 CATALOG일 때만 외부 상세 수집
- 외부 실패 허용: 상세 수집 실패 시 기본 필드만 반환하고 다음 조회에서 재시도
- 도메인 간 협력: bookmarked는 bookmark 도메인, recorded는 record 도메인을 조회해 채운다
- 응답 보정: 원천에 가격이 없으면 컨트롤러가 `"관람료 정보 없음"`으로 채워 내려보낸다(값을 지어내지 않고 부재를 명시)

```mermaid
sequenceDiagram
    actor 사용자
    participant ExhibitionV1Controller
    participant ExhibitionFacade
    participant ExhibitionRepository
    participant Exhibition
    participant CultureExhibitionClient
    participant ExhibitionBookmarkRepository
    participant RecordJpaRepository

    Note right of 사용자: 선택 인증(비로그인 허용)

    사용자->>ExhibitionV1Controller: 전시 상세 조회 요청
    ExhibitionV1Controller->>ExhibitionFacade: 상세 조회

    ExhibitionFacade->>ExhibitionRepository: 전시 조회
    alt 전시가 존재하지 않음
        ExhibitionRepository-->>사용자: 실패
    end

    ExhibitionFacade->>Exhibition: 접근 가능 여부 확인
    alt 타인의 CUSTOM 전시 (403)
        Exhibition-->>사용자: 실패
    end

    alt CATALOG이고 상세 미수집 (detailSyncedAt=null)
        ExhibitionFacade->>CultureExhibitionClient: 외부 상세 1회 지연수집
        alt 상세 수신
            ExhibitionFacade->>Exhibition: 상세 필드 반영 + 동기화 시각 기록
        else 외부 호출 실패
            Note right of CultureExhibitionClient: 기본 필드만 반환 — detailSyncedAt이 null로 남아 다음 조회에서 재시도
        end
    end

    ExhibitionFacade->>Exhibition: 조회수 1 증가
    ExhibitionFacade->>ExhibitionRepository: 저장

    ExhibitionFacade->>ExhibitionBookmarkRepository: 요청자의 북마크 여부 조회
    ExhibitionFacade->>RecordJpaRepository: 요청자의 기록 여부 조회
    Note right of RecordJpaRepository: 비로그인은 조회 없이 둘 다 false

    ExhibitionFacade-->>ExhibitionV1Controller: 결과 반환 (bookmarked·recorded 포함)
    ExhibitionV1Controller-->>사용자: 성공
```
